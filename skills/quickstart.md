# Sol Quickstart

This guide covers the steps to launch `sol` as quickly as possible and verify the `dev -> build -> serve` flow.

## Prerequisites

- Node.js 24+
- `pnpm`
- `moon` / `sol` commands are available

## 1. Create a New Project

```bash
sol new myapp --user yourname
cd myapp
pnpm install
```

Key scripts in the `package.json` generated by `sol new`:

- `dev`: `sol dev`
- `build`: `sol build`
- `serve`: `sol serve`

## 2. Start the Development Server

```bash
# via script
pnpm dev

# direct execution
sol dev
```

Verification points:

- `http://localhost:7777/` is displayed
- `http://localhost:7777/api/health` returns JSON

## 3. Create a Production Build

```bash
# via script
pnpm build

# direct execution
sol build
```

## 4. Serve the Production Build

```bash
# via script
pnpm serve

# direct execution
sol serve
```

## 5. Shortest Flow for Reflecting Changes

When you change routes, verify the diff with the following:

```bash
sol generate --mode dev
sol dev
```

Routing specification source of truth: `docs/routing.md`
When problems occur: `docs/troubleshooting.md`
